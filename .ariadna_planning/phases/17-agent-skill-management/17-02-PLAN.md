**Create action tests (~8-10 tests):**

1. `test "should assign skill to agent"` — POST with skill_id for an unassigned skill, assert_difference("AgentSkill.count", 1), assert redirect to agent show, verify flash notice contains skill name
2. `test "should not duplicate assignment (idempotent)"` — POST with skill_id for @assigned_skill (already assigned via fixture), assert_no_difference("AgentSkill.count"), assert redirect (not error)
3. `test "should not assign skill from another company"` — POST with skill_id for @widgets_skill, assert_response :not_found (Current.company.skills.find will raise RecordNotFound)
4. `test "should not assign skill to agent from another company"` — Switch context: use widgets_agent path but acme session, assert_response :not_found
5. `test "should redirect unauthenticated user on create"` — sign_out, POST, assert_redirected_to new_session_url
6. `test "should redirect user without company on create"` — Create user with no company, sign in, POST, assert_redirected_to new_company_url

**Destroy action tests (~6-8 tests):**

7. `test "should remove skill from agent"` — DELETE existing agent_skill, assert_difference("AgentSkill.count", -1), assert redirect with flash notice
8. `test "should not remove agent_skill belonging to another agent"` — Create agent_skill on http_agent, try to DELETE it via claude_agent's nested route, assert_response :not_found (because @agent.agent_skills.find will fail)
9. `test "should not remove skill from agent in another company"` — Try to DELETE via widgets_agent path, assert_response :not_found
10. `test "should redirect unauthenticated user on destroy"` — sign_out, DELETE, assert_redirected_to new_session_url
