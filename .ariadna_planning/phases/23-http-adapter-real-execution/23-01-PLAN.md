  <action>
Replace the existing HttpAdapter (which only has display_name, description, config_schema) with a full implementation:

**Error classes** (defined inside HttpAdapter):
- `HttpAdapter::PermanentError < StandardError` -- raised for 4xx responses. Message includes HTTP status code and truncated body.
- `HttpAdapter::TransientError < StandardError` -- raised for 5xx responses and Net::HTTP timeout exceptions. Message includes HTTP status code (or timeout description) and truncated body.

**Constants:**
- `OPEN_TIMEOUT = 5` (seconds, connect timeout)
- `READ_TIMEOUT = 30` (seconds, read timeout)
- `MAX_RETRIES = 3` (total attempts for transient failures)
- `BASE_BACKOFF = 1` (seconds, base for exponential backoff: 1s, 2s, 4s)

**Class method `self.execute(agent, context)`:**
1. Extract `url` from `agent.adapter_config["url"]`. Raise `PermanentError, "No URL configured"` if blank.
2. Build payload hash:
   ```ruby
   {
     agent_id: agent.id,
     agent_name: agent.name,
     run_id: context[:run_id],
     trigger_type: context[:trigger_type],
     task: context[:task_id] ? {
       id: context[:task_id],
       title: context[:task_title],
       description: context[:task_description]
     } : nil,
     resume_session_id: context[:resume_session_id],
     delivered_at: Time.current.iso8601
   }.compact
   ```
3. Call `deliver_with_retries(url, payload, agent.adapter_config)` which returns the response.
4. Return result hash: `{ exit_code: 0, response_code: response.code.to_i, response_body: response.body&.truncate(1000) }`

**Private class method `self.deliver_with_retries(url, payload, config)`:**
1. Parse URI from url. Set up `Net::HTTP.new(uri.host, uri.port)`.
2. Configure: `http.use_ssl = (uri.scheme == "https")`, `http.open_timeout = OPEN_TIMEOUT`, `http.read_timeout = READ_TIMEOUT`.
3. Override timeouts from config if present: `config["timeout"]` sets read_timeout (integer, capped at 120s).
4. Build request: `Net::HTTP::Post.new(uri.request_uri)` with headers `{"Content-Type" => "application/json"}`. Merge `config["headers"]` if present. Add `Authorization: Bearer #{config["auth_token"]}` if `config["auth_token"]` present.
5. Set `request.body = payload.to_json`.
6. Retry loop (MAX_RETRIES attempts):
   - `response = http.request(request)`
   - If 2xx (`response.is_a?(Net::HTTPSuccess)`): return response immediately.
   - If 4xx (`response.is_a?(Net::HTTPClientError)`): raise `PermanentError` immediately (no retry). Message: `"HTTP #{response.code}: #{response.body&.truncate(200)}"`.
   - If 5xx (`response.is_a?(Net::HTTPServerError)`) or other non-success: increment attempt counter. If attempts exhausted, raise `TransientError`. Otherwise `sleep(BASE_BACKOFF * (2 ** (attempt - 1)))` and retry.
7. Rescue `Net::OpenTimeout, Net::ReadTimeout, Errno::ECONNREFUSED, Errno::ECONNRESET, SocketError` -- treat as transient. Increment attempt, sleep backoff, retry. On final attempt, raise `TransientError` with the original exception message.

**Key design notes:**
- `PermanentError` propagates up to `ExecuteAgentJob`'s rescue clause, which calls `agent_run.mark_failed!` -- this is correct for 4xx (no retry at job level).
- `TransientError` also propagates up to the same rescue clause after retries are exhausted -- the run is marked failed with the transient error message. This is correct because `ExecuteAgentJob` has `discard_on` (no job-level retry), and the adapter already retried internally.
- Both error classes inherit from `StandardError`, so they are caught by `rescue StandardError, NotImplementedError => e` in ExecuteAgentJob.
- The `require "net/http"` and `require "json"` are not needed -- both are autoloaded in Rails.
