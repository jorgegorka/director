    **Controller (app/controllers/role_templates_controller.rb):**
    Create new controller:

    ```ruby
    class RoleTemplatesController < ApplicationController
      before_action :require_company!
      before_action :set_template, only: [:show, :apply]

      def index
        @templates = RoleTemplateRegistry.all
      end

      def show
      end

      def apply
        result = ApplyRoleTemplateService.call(
          company: Current.company,
          template_key: @template.key
        )

        if result.success?
          redirect_to roles_path, notice: result.summary
        else
          redirect_to role_template_path(@template.key), alert: "Template apply failed: #{result.errors.join(', ')}"
        end
      end

      private

      def set_template
        @template = RoleTemplateRegistry.find(params[:id])
      rescue RoleTemplateRegistry::TemplateNotFound
        raise ActiveRecord::RecordNotFound
      end
    end
    ```

    Key decisions:
    - Uses `params[:id]` as the template key (string-based lookup, not integer ID)
    - Rescues TemplateNotFound and re-raises as RecordNotFound for standard 404 handling
    - `apply` uses the `.call` class method pattern matching existing service usage
    - On success: redirects to roles_path with `notice:` containing result.summary (e.g., "Created 5 roles, skipped 1 existing")
    - On error: redirects back to template show page with `alert:` (per locked decision: standard POST -> redirect)
    - `require_company!` guard follows pattern from RolesController
  </action>

    ```erb
        <div class="hierarchy-tree">
          <%% # Build parent-children map for tree rendering %>
          <%% children_map = @template.roles.group_by(&:parent) %>
          <%% root_roles = @template.roles.select { |r| r.parent.nil? } %>
          <%% render_tree = ->(roles, prefix, is_last_set) do %>
            <%% roles.each_with_index do |role, index| %>
              <%% is_last = index == roles.size - 1 %>
              <%% connector = prefix.empty? ? "" : (is_last ? "&#9492;&#9472;&#9472; " : "&#9500;&#9472;&#9472; ") %>
              <div class="hierarchy-tree__node">
                <span class="hierarchy-tree__connector"><%%= raw(prefix + connector) %></span>
                <div class="hierarchy-tree__role">
                  <strong class="hierarchy-tree__title"><%%= role.title %></strong>
                  <p class="hierarchy-tree__description"><%%= role.description %></p>
                  <%% if role.skill_keys.any? %>
                    <div class="hierarchy-tree__skills">
                      <%% role.skill_keys.each do |skill_key| %>
                        <span class="skill-badge"><%%= skill_key.humanize %></span>
                      <%% end %>
                    </div>
                  <%% end %>
                </div>
              </div>
              <%% child_roles = children_map[role.title] || [] %>
              <%% if child_roles.any? %>
                <%% new_prefix = prefix.empty? ? "" : prefix + (is_last ? "&nbsp;&nbsp;&nbsp; " : "&#9474;&nbsp;&nbsp; ") %>
                <%% render_tree.call(child_roles, new_prefix, nil) %>
              <%% end %>
            <%% end %>
          <%% end %>
          <%% render_tree.call(root_roles, "", nil) %>
        </div>
      </div>
    </section>
    ```

    IMPORTANT implementation notes for the tree:
    - Use HTML entities for tree characters: `&#9500;` = ├, `&#9472;` = ─, `&#9492;` = └, `&#9474;` = │
    - The tree is rendered recursively using a lambda (render_tree) that builds prefix strings for indentation
    - Root roles (parent: nil) start with no prefix
    - Children get prefix with either │ (continuing) or spaces (last child) to draw proper tree lines
    - Each node shows: title (bold), description, and skill badges (reusing existing `.skill-badge` CSS class)
    - The turbo_confirm dialog on the apply button tells the user how many roles will be created
