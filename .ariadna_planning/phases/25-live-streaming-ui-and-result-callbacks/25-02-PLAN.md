<% end %>

**3. Implement broadcast batching in AgentRun (STREAM-05):**

Broadcast batching prevents flooding Action Cable when the adapter is producing output rapidly. The strategy: track the last broadcast time and only broadcast if at least 100ms have elapsed. Otherwise, the line is just persisted (browsers will catch up on next broadcast or page refresh).

Add to `app/models/agent_run.rb`:

```ruby
BROADCAST_MIN_INTERVAL = 0.1 # seconds (100ms) -- STREAM-05

# Class-level tracking for broadcast timestamps per run.
# Thread-safe via the GIL for single-process Puma; good enough for SQLite deployment.
@@last_broadcast_at = {} # rubocop:disable Style/ClassVars

def broadcast_line!(text)
  return if text.blank?
  append_log!(text)

  now = Process.clock_gettime(Process::CLOCK_MONOTONIC)
  last = @@last_broadcast_at[id] || 0

  if (now - last) >= BROADCAST_MIN_INTERVAL
    @@last_broadcast_at[id] = now
    Turbo::StreamsChannel.broadcast_append_to(
      "agent_run_#{id}",
      target: "agent-run-output",
      partial: "agent_runs/log_line",
      locals: { text: text }
    )
  end
end

# Call this when a run reaches terminal state to flush the last broadcast
# and clean up the tracking hash.
def broadcast_flush!
  @@last_broadcast_at.delete(id)
  # Broadcast a complete signal so the UI can update (remove live indicator)
  Turbo::StreamsChannel.broadcast_replace_to(
    "agent_run_#{id}",
    target: "agent-run-live-indicator",
    html: ""
  )
end
```

**1. Add cancel! method to AgentRun model (`app/models/agent_run.rb`):**

```ruby
# Cancels the run: kills the tmux session (if claude_local) and marks as cancelled.
# Called from AgentRunsController#cancel.
def cancel!
  # Kill tmux session if this is a claude_local agent run
  if agent.claude_local?
    session_name = "#{ClaudeLocalAdapter::SESSION_PREFIX}_#{id}"
    ClaudeLocalAdapter.kill_session(session_name)
  end

  mark_cancelled!
  agent.update!(status: :idle) if agent.running?
end
```

This method:
- Kills the tmux session using the same naming convention (`director_run_#{id}`) used by ClaudeLocalAdapter
- Calls the existing `mark_cancelled!` which validates state and sets status + completed_at
- Returns the agent to idle status (matching the pattern in ExecuteAgentJob rescue block)
- Works for non-tmux runs too (HTTP/process) -- just marks cancelled without tmux kill

def cancel

    redirect_to agent_agent_run_path(@agent, @agent_run), alert: "Run is already #{@agent_run.status}."
