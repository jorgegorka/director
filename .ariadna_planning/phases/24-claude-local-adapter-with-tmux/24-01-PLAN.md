**Class method `self.execute(agent, context)`:**
1. **Budget gate (CLAUDE-06):** Check `agent.budget_exhausted?`. If true, raise `BudgetExhausted, "Agent budget exhausted: spent #{agent.monthly_spend_cents} of #{agent.budget_cents} cents budget"`. This prevents any tmux session from spawning.
2. Find the AgentRun via `AgentRun.find(context[:run_id])` to use for log accumulation.
3. Build the tmux session name: `"#{SESSION_PREFIX}_#{context[:run_id]}"`.
4. Build the claude command string via `build_claude_command(agent, context)`.
5. Build the full tmux command: `tmux new-session -d -s #{session_name} -E "#{env_prefix} #{claude_cmd}"` where `env_prefix` sets `ANTHROPIC_API_KEY`.
6. Spawn the tmux session via `system(tmux_spawn_command)`. If system returns false/nil, raise `ExecutionError, "Failed to create tmux session: #{session_name}"`.
7. Poll the session for output and completion via `poll_session(session_name, agent_run)`.
8. After completion, capture final output, parse for result event, clean up tmux session.
9. Return result hash: `{ exit_code: exit_code, session_id: session_id, cost_cents: cost_cents }`.


**Private class method `self.poll_session(session_name, agent_run)`:**
This is the core polling loop that reads tmux output:
1. Initialize `last_line_count = 0`, `accumulated_lines = []`, `poll_count = 0`, `max_polls = (MAX_POLL_WAIT / POLL_INTERVAL).to_i`.
2. Loop:
   a. Check if tmux session still exists: `system("tmux has-session -t #{session_name.shellescape} 2>/dev/null")`. If session is gone, break the loop.
   b. Capture current pane content: `output = `tmux capture-pane -t #{session_name.shellescape} -p -S - 2>/dev/null``. The `-S -` captures from the start of the scrollback buffer.
   c. Split output into lines. If new lines exist beyond `last_line_count`, process only the new ones.
   d. For each new line: `agent_run.append_log!(line + "\n")`, add to `accumulated_lines`.
   e. Update `last_line_count`.
   f. `poll_sleep(POLL_INTERVAL)` -- overridable hook for tests.
   g. Increment `poll_count`. If `poll_count >= max_polls`, kill the session (`system("tmux kill-session -t #{session_name.shellescape} 2>/dev/null")`) and raise `ExecutionError, "Execution timed out after #{MAX_POLL_WAIT} seconds"`.
3. After loop exits (session ended), do one final capture-pane to get any remaining output.
4. Return `accumulated_lines`.

**Private class method `self.parse_result(accumulated_lines)`:**
Parse the accumulated stream-JSON lines to extract session_id and cost:
```ruby
session_id = nil
cost_cents = nil
exit_code = 0

accumulated_lines.each do |line|
  next if line.blank?
  begin
    event = JSON.parse(line)
  rescue JSON::ParserError
    next
  end

  if event["type"] == "result"
    session_id = event["session_id"]  # CLAUDE-03
    if event["total_cost_usd"].present?
      cost_cents = (event["total_cost_usd"].to_f * 100).round  # CLAUDE-05
    end
    if event["subtype"] == "error"
      exit_code = 1
    end
  end
end

{ exit_code: exit_code, session_id: session_id, cost_cents: cost_cents }
```

**Full execute method flow (with error handling):**
```ruby
def self.execute(agent, context)
  raise BudgetExhausted, "Agent budget exhausted: spent #{agent.monthly_spend_cents} of #{agent.budget_cents} cents budget" if agent.budget_exhausted?

  agent_run = AgentRun.find(context[:run_id])
  session_name = "#{SESSION_PREFIX}_#{context[:run_id]}"
  claude_cmd = build_claude_command(agent, context)
  spawn_command = "tmux new-session -d -s #{session_name.shellescape} -E '#{env_prefix} #{claude_cmd}'"

  unless system(spawn_command)
    raise ExecutionError, "Failed to create tmux session: #{session_name}"
  end

  accumulated_lines = poll_session(session_name, agent_run)
  result = parse_result(accumulated_lines)

  result
ensure
  cleanup_session(session_name) if session_name
end
```

**Key design notes:**
- `BudgetExhausted` inherits from StandardError, caught by ExecuteAgentJob rescue clause which calls agent_run.mark_failed! with the budget message.
- `ExecutionError` also inherits from StandardError, caught by same rescue clause.
- The ensure block always cleans up tmux sessions, preventing orphans even on exceptions.
- `--bare` is always passed (CLAUDE-07) -- prevents reading/writing ~/.claude/claude.json, avoiding corruption under concurrency.
- ANTHROPIC_API_KEY is passed via environment prefix in the tmux command, not inherited from the Rails process environment (CLAUDE-07).
- The tmux session provides a real TTY, solving the Claude CLI stdout buffering bug (confirmed upstream issue #25670).
- `shellescape` is used on all user-supplied values to prevent shell injection.
