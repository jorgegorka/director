    Create `app/controllers/org_charts_controller.rb`:
    ```ruby
    class OrgChartsController < ApplicationController
      before_action :require_company!

      def show
        @roles = Current.company.roles.includes(:parent, :children).order(:title)
        @root_roles = @roles.select(&:root?)
      end
    end
    ```

    Key points:
    - Loads all roles for the company with eager loading of parent and children to avoid N+1
    - Separates root roles for the tree rendering entry points
    - Uses `.select(&:root?)` on the already-loaded collection rather than an additional query

    **Step 3: Create org charts helper**

    Create `app/helpers/org_charts_helper.rb`:
    ```ruby
    module OrgChartsHelper
      def roles_tree_data(root_roles)
        root_roles.map { |role| role_node_data(role) }
      end

      private

      def role_node_data(role)
        {
          id: role.id,
          title: role.title,
          description: role.description.to_s.truncate(80),
          url: role_path(role),
          agent_name: nil,  # Placeholder for Phase 4
          children: role.children.order(:title).map { |child| role_node_data(child) }
        }
      end
    end
    ```

    This recursively builds the tree as a nested JSON structure that the Stimulus controller consumes.

    **Step 1: Create the org chart Stimulus controller**

    Create `app/javascript/controllers/org_chart_controller.js`:

    This controller reads the tree data from the `roles` value, calculates a top-down tree layout, and renders:
    - SVG `<foreignObject>` elements for each role node (containing HTML role cards)
    - SVG `<path>` elements for connecting lines between parent and child nodes

    Layout algorithm overview:
    - **Top-down tree layout**: Roots at the top, children below
    - **Bottom-up width calculation**: Each node's subtree width is the max of its own width or the sum of children's subtree widths plus gaps
    - **Top-down position assignment**: Each node is horizontally centered above its subtree
    - **Multiple roots supported**: Root nodes are arranged left-to-right at the top level
    - **Curved bezier connections**: Cubic bezier curves from parent bottom-center to child top-center with a vertical midpoint

    Constants: NODE_WIDTH=220, NODE_HEIGHT=100, HORIZONTAL_GAP=40, VERTICAL_GAP=80, PADDING=40

    Implementation details:

    The `connect()` lifecycle method calls `render()`.

    The `render()` method:
    1. Clears the SVG element
    2. Calls `calculateLayout(this.rolesValue)` to get node positions and connection coordinates
    3. Sets SVG width/height/viewBox from layout bounds
    4. Iterates connections and calls `drawConnection(svg, conn)` for each
    5. Iterates nodes and calls `drawNode(svg, node)` for each

    The `calculateLayout(roots)` method:
    1. Bottom-up pass: recursively calculates `_subtreeWidth` for each node. Leaf nodes have width = NODE_WIDTH. Internal nodes have width = max(NODE_WIDTH, sum of children subtree widths + gaps).
    2. Top-down pass: assigns (x, y) coordinates. Each node is centered above its subtree. Children are laid out left-to-right within the parent's subtree allocation.
    3. Collects all nodes (with x, y, width, height, title, url, description, agentName) and connections (with x1, y1, x2, y2) into arrays.
    4. Returns { nodes, connections, width, height } where width/height include padding.

    The `drawConnection(svg, conn)` method:
    - Creates an SVG `<path>` element with namespace "http://www.w3.org/2000/svg"
    - Calculates midY = (conn.y1 + conn.y2) / 2
    - Sets `d` attribute to `M x1 y1 C x1 midY, x2 midY, x2 y2` (cubic bezier)
    - Sets class to "org-chart-line", fill to "none"
    - Appends to SVG

    The `drawNode(svg, node)` method:
    - Creates an SVG `<foreignObject>` element with namespace "http://www.w3.org/2000/svg"
    - Sets x, y, width, height attributes from node data
    - Creates DOM elements using safe DOM methods (createElement, textContent, appendChild) to prevent XSS:
      - Creates a wrapper `<div class="org-chart-node">`
      - Creates an `<a>` element with href=node.url, class="org-chart-node__link", data-turbo-frame="_top"
      - Creates a `<span class="org-chart-node__title">` and sets its textContent to node.title
      - Creates a `<span class="org-chart-node__agent">` containing a dot span and text node for agent name (or "Unassigned")
      - If node.description exists, creates a `<span class="org-chart-node__desc">` with textContent
      - Assembles the DOM tree by appending children to parents
    - Appends foreignObject to SVG

    IMPORTANT: All user-provided text (title, description, agent_name) MUST be set via `element.textContent = value` (never via innerHTML or string interpolation into HTML) to prevent XSS. Build the DOM tree programmatically using createElement/createTextNode/appendChild.
