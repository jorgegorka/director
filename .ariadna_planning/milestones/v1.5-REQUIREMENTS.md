# Requirements Archive: v1.5 Role Templates

**Archived:** 2026-03-30
**Status:** All 13 requirements complete

## Template Data

- [x] **TMPL-01**: App ships with 5 builtin department YAML templates (Engineering, Marketing, Operations, Finance, HR) each defining a role hierarchy with titles, descriptions, job specs, and skill key references
- [x] **TMPL-02**: RoleTemplateRegistry class loads, caches, and exposes all YAML template definitions with find-by-key access
- [x] **TMPL-03**: Each template defines 4-7 roles with parent references in dependency order and 3-5 skill assignments per role

## Application Service

- [x] **APPLY-01**: ApplyRoleTemplateService creates roles in dependency order (parents before children) with correct hierarchy
- [x] **APPLY-02**: Service skips roles whose title already exists in the company
- [x] **APPLY-03**: Service pre-assigns skills from the company's skill library to each created role
- [x] **APPLY-04**: Service returns a structured result with created/skipped/errors counts
- [x] **APPLY-05**: Apply All action applies all 5 department templates in sequence under a shared CEO

## Templates UI

- [x] **UI-01**: Dedicated templates browse page lists all available department templates as cards with name, description, and role count
- [x] **UI-02**: Template detail page shows the full role hierarchy tree with descriptions and skill badges
- [x] **UI-03**: One-click apply button creates the department and redirects with flash summary (created X, skipped Y)
- [x] **UI-04**: Link from roles index page to templates browse page

## Skill Mappings

- [x] **SKILL-01**: Extended default_skills.yml with 18 new role-title-to-skill mappings for template roles (total: 29)
