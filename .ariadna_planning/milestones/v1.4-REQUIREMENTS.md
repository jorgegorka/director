# Requirements Archive: v1.4 Agent Execution

**Archived:** 2026-03-28
**Status:** All 26 requirements complete

## Execution Data Model

- [x] **EXEC-01**: AgentRun model with state machine (queued -> running -> completed/failed/cancelled)
- [x] **EXEC-02**: AgentRun stores accumulated log text, exit code, cost_cents, and timing
- [x] **EXEC-03**: AgentRun stores claude_session_id for session resumption across runs
- [x] **EXEC-04**: ExecuteAgentJob dispatches to correct execution service based on adapter_type
- [x] **EXEC-05**: Agent status transitions (idle -> running -> idle/error) reflect real execution state
- [x] **EXEC-06**: Dedicated Solid Queue `execution` queue isolates long-running jobs from default queue

## HTTP Adapter

- [x] **HTTP-01**: WakeAgentService delivers real HTTP POST to agent's configured URL with task context payload
- [x] **HTTP-02**: HTTP delivery classifies 4xx as permanent failure and 5xx/timeout as transient (retryable)
- [x] **HTTP-03**: HTTP delivery retries transient failures with exponential backoff
- [x] **HTTP-04**: HTTP delivery uses explicit timeouts (open: 5s, read: 30s)

## Claude Local Adapter

- [x] **CLAUDE-01**: Tmux-based session management -- spawns `claude -p --bare --output-format stream-json` in a named tmux session
- [x] **CLAUDE-02**: Stream-json output parsed line-by-line, accumulated in AgentRun log
- [x] **CLAUDE-03**: Session ID captured from result event and stored in AgentRun
- [x] **CLAUDE-04**: Subsequent runs for same agent pass `--resume <session_id>` for conversation continuity
- [x] **CLAUDE-05**: `total_cost_usd` from result event written to AgentRun cost_cents and fed into budget tracking
- [x] **CLAUDE-06**: Execution blocked when agent.budget_exhausted? returns true
- [x] **CLAUDE-07**: `ANTHROPIC_API_KEY` passed explicitly via environment, `--bare` flag mandatory

## Live Streaming UI

- [x] **STREAM-01**: AgentRun show view streams live output via turbo_stream_from while agent is active
- [x] **STREAM-02**: Agent status broadcast on agent-scoped stream updates dashboard
- [x] **STREAM-03**: Tool-use indicators parsed from stream-json content_block_start events and displayed
- [x] **STREAM-04**: Cancel button kills tmux session and marks AgentRun as cancelled
- [x] **STREAM-05**: Broadcast batching (minimum 100ms interval) prevents Action Cable flooding

## Result Callbacks

- [x] **CALLBACK-01**: `POST /api/agent_runs/:id/result` endpoint for agents to report task completion
- [x] **CALLBACK-02**: `POST /api/agent_runs/:id/progress` endpoint for agents to report intermediate progress
- [x] **CALLBACK-03**: Result callback updates task status and posts completion message to task conversation
- [x] **CALLBACK-04**: Cost reporting via result callback feeds into agent budget tracking
