# Requirements Archive: v1.3 Agent Hooks

**Archived:** 2026-03-28
**Status:** All 15 requirements complete

## Data Model

- [x] **DATA-01**: AgentHook belongs to agent with lifecycle_event, action_type (trigger_agent/webhook), action_config, enabled flag, and position ordering
- [x] **DATA-02**: HookExecution tracks each hook firing with status (queued/running/completed/failed), input/output payloads, timing, and error messages
- [x] **DATA-03**: Agent has_many agent_hooks with dependent destroy

## Hook Triggering

- [x] **TRIG-01**: Hookable concern detects task status transitions and enqueues matching hooks as background jobs
- [x] **TRIG-02**: Only enabled hooks for the task's assignee fire, ordered by position
- [x] **TRIG-03**: Completed subtasks with a parent_task enqueue ProcessValidationResultJob

## Hook Actions

- [x] **ACT-01**: trigger_agent action creates a validation subtask assigned to target agent and wakes it via WakeAgentService
- [x] **ACT-02**: webhook action POSTs JSON payload to configured URL with custom headers and timeouts
- [x] **ACT-03**: ExecuteHookJob retries on failure with polynomial backoff (3 attempts)

## Feedback Loop

- [x] **FEED-01**: ProcessValidationResultService collects validation subtask messages and posts feedback on the parent task
- [x] **FEED-02**: Original agent is woken with review_validation context after feedback is posted
- [x] **FEED-03**: Audit events recorded for hook_executed and validation_feedback_received

## Management UI

- [x] **UI-01**: AgentHooksController provides CRUD for hooks nested under agents
- [x] **UI-02**: Company scoping ensures hooks are only accessible within the owning company
- [x] **UI-03**: HeartbeatEvent trigger_type extended with hook_triggered
