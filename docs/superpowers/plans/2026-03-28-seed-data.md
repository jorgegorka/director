# Seed Data Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace `db/seeds.rb` with a comprehensive seed script that creates a self-referential "Director AI" company with 12 agents, full org chart, goal tree, 32 tasks, messages, approval gates, audit events, and notifications.

**Architecture:** Single `db/seeds.rb` file, top-to-bottom Ruby script. Wipes existing data, then creates all records using `create!` calls. Company creation auto-seeds 50 builtin skills via `after_create :seed_default_skills!` callback. Role creation with agent auto-assigns default skills via `after_save :assign_default_skills_to_agent` callback when title matches `config/default_skills.yml` keys.

**Tech Stack:** Rails 8.1, SQLite, Minitest fixtures (unaffected — seeds are separate)

**Spec:** `docs/superpowers/specs/2026-03-28-seed-data-design.md`

### Task 3: Role Hierarchy & Skill Auto-Assignment

**Files:**
- Modify: `db/seeds.rb`

Role titles must match `config/default_skills.yml` keys (case-insensitive) for auto-skill assignment. The YAML has keys: `ceo`, `cto`, `cmo`, `engineer`, `designer`, `pm`, `qa`, `devops`, `researcher`, `general`. The `after_save :assign_default_skills_to_agent` callback fires on first agent assignment.

Strategy:
- Create roles WITHOUT agents first (to build the hierarchy)
- Then update each role with its agent (triggers `first_agent_assignment?` → auto-skills)
- For roles whose titles don't match YAML keys (Backend Engineer, Frontend Engineer, Security Engineer, SEO Specialist, Content Strategist), manually assign skills after

- [ ] **Step 2: Append manual skill assignments for non-matching roles**

The titles "Security Engineer", "SEO Specialist", and "Content Strategist" don't match any `default_skills.yml` key, so they get no auto-skills. Assign manually:

```ruby
# Manual skill assignments for roles that don't match default_skills.yml keys
manual_skill_assignments = {
  "Security Engineer"  => %w[security_assessment code_review risk_assessment monitoring_alerting incident_response],
  "SEO Specialist"     => %w[content_strategy market_analysis data_analysis audience_research report_writing],
  "Content Strategist" => %w[content_strategy brand_management audience_research documentation communication]
}

manual_skill_assignments.each do |agent_name, skill_keys|
  agent = agents[agent_name]
  skills_to_assign = company.skills.where(key: skill_keys)
  skills_to_assign.each do |skill|
    agent.agent_skills.find_or_create_by!(skill: skill)
  end
end

puts "  Manually assigned skills to Security Engineer, SEO Specialist, Content Strategist"
```

def create_task!(attrs)
  task = Task.create!(attrs)
  if task.completed? && attrs[:completed_ago]
    task.update_columns(completed_at: attrs[:completed_ago].ago, created_at: (attrs[:completed_ago] + 2.days).ago)
  end
  task
